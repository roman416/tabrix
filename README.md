# Table OS

Прототип табличного веб-приложения на Flask для загрузки и анализа CSV, TSV, XLSX, XLSM, XLS и Parquet.

## Что уже работает

- импорт таблиц
- нормализация любой загруженной таблицы в UTF-8 CSV-кэш для внешних C/EXE-инструментов
- просмотр и ручное редактирование ячеек
- добавление и удаление строк
- фильтр по таблице и пагинация предпросмотра
- статистика по колонкам
- построение сводной таблицы
- сортировка с выгрузкой результата в CSV
- автодашборды по данным
- локальный помощник через Ollama с fallback-ответом на Python

## Внешние инструменты C / EXE

Проект теперь работает через внешние бинарники:

- `native_tools/windows/sort.exe`
- `native_tools/windows/sred.exe`
- `native_tools/linux/sort`
- `native_tools/linux/sred`

Логика такая:

- при загрузке исходный файл сохраняется как есть
- параллельно создаётся нормализованный CSV-кэш в `uploads/_csv_cache/<table_id>.csv`
- сортировка идёт по этому CSV через `sort` / `sort.exe`
- числовые базовые метрики `min`, `max`, `mean`, `sum` идут через `sred` / `sred.exe`
- остальные поля статистики добираются сервером из того же CSV-кэша, чтобы интерфейс не развалился

## Почему сделано так

Ваши C-инструменты работают только с CSV, а Excel-файлы любят ломать ожидания. Поэтому после загрузки таблица приводится к плоскому UTF-8 CSV без переносов строк внутри ячеек. Так внешний бинарник и веб-интерфейс смотрят на один и тот же набор данных, а не на две параллельные вселенные.

Для `.xlsx` и `.xlsm` чтение сделано через `openpyxl` с `data_only=True`, а не через обычный `pandas.read_excel()`. Это обычно стабильнее для значений ячеек. Для `.xls` нужен `xlrd`.

## Запуск

### Linux / macOS

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

### Windows

```bat
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

После запуска откройте:

```text
http://127.0.0.1:5000
```

## Ollama

По умолчанию ожидается:

- URL: `http://127.0.0.1:11434`
- модель: `deepseek-r1:8b`

Можно переопределить через переменные окружения:

```bash
export OLLAMA_URL=http://127.0.0.1:11434
export OLLAMA_MODEL=deepseek-r1:8b
```

Windows:

```bat
set OLLAMA_URL=http://127.0.0.1:11434
set OLLAMA_MODEL=deepseek-r1:8b
```

## Замечания

- сортировка не перезаписывает исходную таблицу, а отдаёт новый CSV на скачивание
- если внешний бинарник недоступен, сортировка и статистика автоматически откатываются на Python fallback
- `sred.exe` в текущем виде возвращает только `min max mean sum`, поэтому медиана, дисперсия, стандартное отклонение и списки уникальных значений собираются сервером из того же CSV-кэша
- CSV-кэш специально убирает переносы строк внутри ячеек, потому что ваш C-парсер читает файл построчно
- состояние хранится в памяти процесса и в загруженных файлах, без многопользовательской синхронизации
